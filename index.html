My first project involved creating a hospital management system, which made me realize the importance of properly structured tables and relationships. I encountered issues with redundant data and slow queries, which I resolved by normalizing the database and adding indexes where needed.

In subsequent projects, I incorporated best practices such as separating user data into lookup tables, enforcing foreign key constraints, and using transactions for data integrity.

  • Normalization: Prevented duplicate and inconsistent data through 3NF design.
  • Query Optimization: Learned to write efficient queries and use EXPLAIN to analyze performance.
  • Security: Applied parameterized queries to protect against SQL injection.
